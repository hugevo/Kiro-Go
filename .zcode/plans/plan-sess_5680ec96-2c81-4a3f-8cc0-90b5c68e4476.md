## Mục tiêu
Cho phép quản lý từ UI admin: (1) map Kiro version → build hash, (2) 3 version field (KiroVersion/SystemVersion/NodeVersion). Áp dụng đúng các pattern hiện có trong codebase. Không phá backward-compat (config cũ vẫn chạy).

## Thiết kế quan trọng (2 quyết định đã chốt)
- **Scope**: 1 card mới gộp cả map version→hash + 3 version field.
- **Quan hệ với code**: UI override + giữ default. Map hardcode `kiroBuildHashes` trong code làm default không thể xóa. UI lưu override riêng, `ResolveKiroBuildHash` check UI trước → code default → fallback sha256.

## Tóm tắt luồng dữ liệu
```
UI card → POST /admin/api/kiro-client
       → handler apiUpdateKiroClientConfig
       → config.UpdateKiroClientConfig(versions, hashOverrides)
       → Save() → data/config.json
ResolveKiroBuildHash(ver, account.HashSuffix):
  account override → cfg.KiroBuildHashOverrides[ver] → code default kiroBuildHashes → sha256 fallback
```

## Chi tiết implement theo file

### 1. `config/config.go` — backend layer
- **Thêm field Config**: `KiroVersion`/`SystemVersion`/`NodeVersion` đã có sẵn (line 210-212), chỉ cần thêm `KiroBuildHashOverrides map[string]string \`json:"kiroBuildHashes,omitempty"\`` cho user override.
- **Đổi `ResolveKiroBuildHash`** (line 60): thêm 1 nhánh — sau `account.HashSuffix` override, trước code default, check `cfg.KiroBuildHashOverrides[kiroVersion]`. Thứ tự ưu tiên: account override → user UI override → code default → sha256 fallback.
- **Thêm config helper** theo pattern `UpdatePreferredEndpoint` (lock + set field + Save):
  - `GetKiroClientSettings()` — trả về `{kiroVersion, systemVersion, nodeVersion, buildHashes (merge default+override), defaults (hash gốc không xóa được)}`. Read-only, RLock.
  - `UpdateKiroClientSettings(kiroVer, sysVer, nodeVer string, hashOverrides map[string]string)` — Lock + set 3 version field + override map + Save.
- **Giữ nguyên** `kiroBuildHashes` var (line 51) làm default + `stableFallbackHash`.

### 2. `proxy/handler.go` — REST endpoint
- **Thêm route** trong switch `handleAdminAPI` (sau `/prompt-filter`, ~line 2294):
  ```go
  case path == "/kiro-client" && r.Method == "GET":  h.apiGetKiroClientConfig(w, r)
  case path == "/kiro-client" && r.Method == "POST": h.apiUpdateKiroClientConfig(w, r)
  ```
- **Thêm 2 handler** theo pattern `apiGetEndpointConfig`/`apiUpdateEndpointConfig` (line 3824):
  - `apiGetKiroClientConfig` — `json.Encode(config.GetKiroClientSettings())`.
  - `apiUpdateKiroClientConfig` — decode `{kiroVersion, systemVersion, nodeVersion, buildHashes}`, validate (hash phải là 64-hex nếu khác rỗng), gọi `config.UpdateKiroClientSettings`. Trả `{success:true}` hoặc `{error}`.

### 3. `web/index.html` — card UI mới
Thêm 1 card mới trong `#tabSettings` (sau prompt filter card, ~line 410), cấu trúc giống prompt filter card:
- 3 input text: `#kiroVersion`, `#systemVersion`, `#nodeVersion` (kèm hint hiện giá trị default).
- Container `<div id="kiroBuildHashes">` render dynamic rows.
- Nút "Add version" (`#addBuildHashBtn`) + nút Save (`#saveKiroClientBtn`).
- Mỗi row: input version + input hash (64-hex) + nút xóa (trừ default row disable xóa, có badge "default").
- `data-i18n` keys cho mọi label.

### 4. `web/app.js` — logic
Mirror prompt filter pattern:
- Module var `kiroBuildHashes = []` (array object `{version, hash, isDefault}`).
- `loadKiroClientConfig()` — GET, fill 3 version input + `kiroBuildHashes`, gọi `renderKiroBuildHashes()`.
- `saveKiroClientConfig()` — POST `{kiroVersion, systemVersion, nodeVersion, buildHashes}`.
- `renderKiroBuildHashes()` — innerHTML per row, default row có badge + disable remove.
- `addBuildHashRow()` — push `{version:'', hash:'', isDefault:false}` + re-render.
- Event delegation trên `#kiroBuildHashes`: `input` → mutate array theo `data-hash-idx`/`data-hash-field`; `click [data-hash-remove]` → splice (block nếu isDefault).
- Đăng ký: `loadKiroClientConfig()` vào `Promise.all` ở `loadSettings()` (line 1540); `saveKiroClientBtn`/`addBuildHashBtn` vào `bindSettingsEvents()` (line 2992); event delegation cho rows trong cùng `bindSettingsEvents` hoặc binder riêng.

### 5. `web/locales/en.json` + `zh.json` — i18n

## Backward-compat
- Config cũ (không có `kiroBuildHashes` field) → vẫn chạy, `KiroBuildHashOverrides` nil → fallback code default.
- Hash default `2ecd375f...` cho 0.12.333 luôn có sẵn kể cả khi UI trống.
- 3 version field đã tồn tại trong struct, chỉ thêm UI exposure, không breaking.
